# 04 · 接手開發說明（給工程師 / 給 AI）

## 專案概觀

- **用途**：IT 系統整合商 / 資料中心服務商 / 小型 AI 運算建置商的搜尋與評分資料庫。
- **技術**：.NET 8、ASP.NET Core MVC、EF Core、SQLite、Bootstrap 5、ClosedXML。
- **狀態**：MVP 可執行；本機/區網可跑。詳見 `docs/02-開發紀錄與決策.md`。

## 專案結構

```
PartnerFinder/
  Program.cs                 啟動、DI 註冊、啟動時自動 migrate + seed
  appsettings.json           SQLite 連線字串、Brave 金鑰（Search:Brave:ApiKey）
  Models/
    Partner.cs               唯一核心實體（全部欄位）
    Enums.cs                 PartnerStatus / RecommendedLevel / ManualReviewStatus
    ViewModels.cs            Dashboard、Partner 清單用的 ViewModel
  Data/
    AppDbContext.cs          EF Core DbContext
    PartnerOptions.cs        Service Category 下拉選單清單（要改分類改這）
    DbSeeder.cs              首次啟動的範例資料
  Services/
    ScoringService.cs        100 分評分模型（每項都有註解）
    KeywordGeneratorService.cs   關鍵字產生（免 API）
    ExportService.cs         CSV + Excel 匯出（欄位單一來源）
    DuplicateDetectionService.cs 名稱+網域重複偵測
    Connectors.cs            未來連接器「介面」+ no-op 實作
    BraveWebSearchConnector.cs   IWebSearchConnector 的 Brave 實作
  Controllers/               Home, Partners, Keyword, WebSearch, Qualification, Export, Settings
  Views/                     Razor + Bootstrap
  Migrations/                EF Core migration（InitialCreate）
```

## 建置 / 執行 / 測試

```bash
cd PartnerFinder
dotnet build
dotnet run --urls "http://0.0.0.0:5080"     # 區網可連；本機用 http://localhost:5080
```
- 啟動時 `Program.cs` 會 `db.Database.Migrate()` 自動建/升級 DB，並在空庫時 seed 範例資料。
- 資料庫檔 `partnerfinder.db`（已被 `.gitignore` 排除，不進版控）。

## 修改 EF 資料結構（改欄位時）

```bash
dotnet tool install --global dotnet-ef --version 8.0.11   # 一次
dotnet ef migrations add <名稱>
# 下次啟動會自動套用；或手動 dotnet ef database update
```

## 慣例 / 設計要點

- **評分**集中在 `ScoringService.Score()`，回傳逐項 `ScoreLine` 供詳細頁顯示；
  `Apply()` 會把 Total / AiScore / Level 寫回實體。存檔前一定呼叫 `Apply()`。
- **匯出欄位**只在 `ExportService.Columns` 定義一次，CSV 與 Excel 共用。
- **列舉存字串**：`AppDbContext` 用 `HasConversion<string>()`，方便直接讀 `.db` 檔。
- **設定驅動的連接器**：金鑰有值才 `IsConfigured==true`；UI 依此顯示啟用與否。

## 如何新增一個「未來連接器」（例如 SerpAPI）

1. 在 `Services/Connectors.cs` 已有介面（如 `ISerpApiConnector`）。
2. 新增實作類別（參考 `BraveWebSearchConnector`），從 `IConfiguration` 讀金鑰。
3. 在 `Program.cs` 把對應的 `Null...` 註冊換成你的實作
   （需 HttpClient 就用 `builder.Services.AddHttpClient<介面, 實作>()`）。
4. `Settings` 頁會自動反映 `IsConfigured` 狀態。

## 目前待辦（建議優先序）

1. **手機/區網連線**：屬環境問題（防火牆 / IP / Wi-Fi 隔離），非程式問題。見 `02` 文件。
2. **雲端部署**（若要外網可用）：加 `Dockerfile`，部署到 Render / Railway / Azure App Service；
   資料若要持久化與多人共用，改用 PostgreSQL（EF 換 `UseNpgsql`）。
3. 實作其餘連接器：SerpAPI、Microsoft Partner directory、AI Summary 產生器、Weekly market radar。
4. AI Summary：可接 Claude API（Anthropic）產生 `AiSummary` / `AiInfrastructureSummary`。
