# 02 · 開發紀錄與決策

## 一句話總結

已交付一套**可執行的 .NET 8 MVP**：ASP.NET Core MVC + EF Core + SQLite + Bootstrap，
完全本機執行、離線可用；需求裡的功能都做了，並額外加上「自動網路搜尋（Brave API）」。

---

## 技術選型與理由

| 項目 | 決定 | 理由 |
|---|---|---|
| 框架 | ASP.NET Core **MVC**（非 Blazor） | 對非資深維護者較直覺、資料多、範例多 |
| 資料庫 | **SQLite**（單一 `.db` 檔） | 免安裝資料庫伺服器，複製檔案即備份 |
| ORM | **EF Core** + Migration | 自動建表、日後改結構有版本 |
| UI | **Bootstrap 5（打包在本機 `wwwroot/lib`）** | 離線也能用，不依賴 CDN |
| Excel 匯出 | **ClosedXML** 套件 | 產生原生 `.xlsx`，穩定 |
| 付費 API | 初版**不接**；預留介面 | 遵循需求「先本地 MVP」 |
| 自動搜尋 | 後來加上 **Brave Search API**（使用者選定） | 免費額度夠、只需一把金鑰 |

## 已完成功能

- **資料模型**：單一 `Partner` 實體，含需求列出的所有欄位（基本 / IT-DC / AI / 評估）。
- **CRUD**：新增、編輯、刪除、詳細頁；清單頁支援搜尋 + 篩選（文字 / 國家 / 分類 / 等級 / 只看 AI）。
- **評分引擎**（`Services/ScoringService.cs`）：100 分模型，存檔時自動計算分數與 A/B/C 分級，
  詳細頁顯示逐項分數細目。
- **Search Keyword Generator**：依 Country/City/Service Type/AI 產生可複製的搜尋詞（免 API）。
- **自動網路搜尋（Web Search 頁）**：Brave API，輸入詞→列出結果→一鍵「File as partner」帶入建檔。
- **AI Summary（Claude API）**（`ClaudeAiSummaryService`，官方 Anthropic C# SDK）：Detail 頁一鍵
  產生評估摘要 / AI 基礎設施摘要 / 建議補勾能力（僅供人工確認，不自動勾選）/ 建議下一步；
  金鑰設定於 `Ai:Anthropic:ApiKey`，模型可換（預設 claude-opus-4-8）。見 `AI摘要設定.md`。
- **公司資料自動抓取**（`WebsiteInfoService`）：按「File as partner」時自動讀取該公司網站
  （首頁＋聯絡頁，最多 2 個請求），抓出公司名、Email、電話、LinkedIn、美國城市/州、
  服務描述並預填表單；抓不到的欄位留白由人工補。屬 best-effort（網頁上有公開才抓得到）。
- **Dashboard**：總數、A/B/C 數、各國家分布、AI 能力數、缺聯絡資訊數、待人工複核數。
- **匯出**：CSV（含 UTF-8 BOM，Excel 開中文不亂碼）與 Excel `.xlsx`。
- **重複偵測**（`DuplicateDetectionService`）：以正規化公司名 + 網站網域比對，新增時提示疑似重複。
- **未來連接器介面**（`Services/Connectors.cs`）：Web Search / SerpAPI / MS Partner / AI Summary /
  Weekly market radar 都預留介面；Web Search 已實作（Brave）。
- **資料庫初始化**：EF Core `InitialCreate` migration；程式啟動時自動建/升級資料庫並塞入
  3 筆美國範例資料（`Data/DbSeeder.cs`）。
- **驗證**：實際 build（0 warning / 0 error）、實際執行、8 個頁面 + 匯出皆回 HTTP 200；
  評分正確（範例 Summit = 86 分 → A 級）；Brave 連線路徑以假金鑰驗證過（請求/標頭正確）。

## 使用者互動過程中做的補強

1. **手機檢視**：提供手機視角截圖讓使用者先看成果。
2. **區網執行**：加入 `run-on-lan.bat` / `run-on-lan.sh`，讓 app 綁 `0.0.0.0:5080`，
   手機同 Wi-Fi 可連（預設 `dotnet run` 只綁 localhost）。
3. **HTTPS 調整**：`Program.cs` 只在 Production 才強制 HTTPS，本機用 http 不會被導轉、console 乾淨。
4. **自動搜尋**：依使用者選擇實作 **Brave Search API** 連接器 + Web Search 頁 + 設定教學。

## ⚠️ 目前已知 / 待處理

- **手機連不到本機 app（進行中）**：使用者電腦 `localhost:5080` 正常，但用區網 IP
  `192.168.100.209:5080` 連不到。已排除步驟：
  1. 確認啟動時顯示 `Now listening on: http://0.0.0.0:5080`。
  2. Windows 防火牆放行 5080（需「**以系統管理員身分**」執行 PowerShell，否則會「存取被拒」）：
     `New-NetFirewallRule -DisplayName "PartnerFinder 5080" -Direction Inbound -Protocol TCP -LocalPort 5080 -Action Allow`
  3. **懷疑點**：`192.168.100.209` 可能不是實際在用的網卡 IP（電腦若裝 VMware/VirtualBox/WSL/VPN
     會出現假的 192.168.x.x）。應以 `ipconfig` 找「無線區域網路介面卡 Wi-Fi / 乙太網路」底下的 IPv4。
  4. 其他可能：手機不同 Wi-Fi、或 Wi-Fi 開了「裝置隔離」。
  → 這是**環境設定問題，不是程式問題**（程式本機完全正常）。

- **Brave 金鑰尚未申請**：程式已就緒，待使用者申請免費金鑰貼入 `appsettings.json` 即可用
  （見 `自動搜尋設定.md`）。
